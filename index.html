CalmAnchor

A quiet digital companion for CPTSD self-management.

CalmAnchor transforms Kimberley Hajee's 120-page peer-created CPTSD Toolkit into an interactive, accessible web and mobile application. It does not replace the workbook — it makes it usable in real time, when the user is dysregulated, overwhelmed, or in crisis. All data stays local. The user is always in control.

✦ Trauma-informed ✦ Single-user, private ✦ Offline-first No AI No biometrics No data sharing

02 — Scope

The Core 7 + 2

The MVP consists of seven core features and two supporting features. Every feature maps directly to a chapter or exercise in Kimberley's Toolkit. The app does not invent content.

01
Interactive Toolkit Browser
Core

Embedded PDF viewer with chapter navigation, bookmarks, and an in-app search. PDF is bundled for offline reading.

02
Private Reflection Diary
Core

A safe, private space to journal with three gentle prompts. Optional nervous-system-state tagging. No streaks, no pressure.

03
Self-Soothing Exercise Menu
Core

Digitised exercises across five categories: Body, Breath, Voice, Mind, Senses — plus somatic therapy techniques. Guided timers and audio cues.

04
Crisis Grounding Widget
Core

One-tap, always-accessible grounding. Offers 5-4-3-2-1 sensory grounding, box breathing, and a user-curated Quick Reset. Crisis contacts always visible.

05
Manual Trigger & State Tracker
Core

User-initiated logging of triggers, context, and survival responses (Fight, Flight, Freeze, Fawn). No automatic detection. All fields optional.

06
User Portfolio
Core

A personal "comfort box" of favourited exercises, safe-space notes, and custom strategies. Reorderable, categorised, fully user-controlled.

07
Pattern Dashboard
Core

Gentle, descriptive charts from manual logs only — trigger frequency, response distribution, exercise effectiveness, time-of-day patterns. Never diagnostic.

08
Onboarding Workflow
Supporting

A warm, skippable introduction to the app and the toolkit. Every step can be skipped. No registration, no personal data collected.

09
Simple Data Export
Supporting

User-initiated CSV or Excel export of diary, trigger logs, and portfolio. Local-only. No cloud upload. The user's data is theirs to take.

04 — Technical

Technical Architecture

A pragmatic, single-user stack. The app is a Progressive Web App with a local-first data model — everything works offline, especially the Crisis Widget. No cloud sync by default.

Frontend · Next.js + TypeScript
React-based PWA. Single-page app structure, installable on iOS and Android.
Styling · Tailwind CSS
Configured against the existing design system tokens. CSS custom properties remain the single source of truth.
Database · PostgreSQL (Supabase)
Available for opt-in backup/sync only. Local storage (IndexedDB / SQLite) is the primary store.
Auth · Supabase Auth
Used only for encrypted cloud backup if the user explicitly opts in. App is anonymous by default.
PDF · PDF.js (bundled)
Open-source viewer. Toolkit PDF bundled with the app for full offline access.
Export · Client-side CSV/Excel
Generated in the browser. No server, no API. User keeps full control of the file.
Hosting · Vercel / static
Fast, cost-effective hosting for the PWA. This documentation site lives on GitHub Pages.
Mobile · PWA install
Cross-platform (iOS, Android, desktop). One codebase, no app store gatekeeping for the MVP.
Privacy · Local-first
All data on-device. No external analytics. No tracking. User can wipe everything in one tap.
